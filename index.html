Configuraciones Recomendadas

⚙️ Configuración del Buffer RFID

  • Intervalo de envío: 10 segundos (ajustar según volumen de lecturas)
  • Máximo de tags por envío: 50 tags (incrementar si hay alta densidad)
  • Historial de lecturas: 100 registros (suficiente para auditoría)

📡 Distancias de Lectura Recomendadas

Distancia Potencia Uso Recomendado
10-49 cm 5 dBm Control individual, escritorio
50-99 cm 5-7 dBm Lectura de documentos cercanos
1-2 metros 15-18 dBm RECOMENDADO - Lectura rápida y precisa
3-4 metros 18-22 dBm Alcance medio-corto
5-6 metros 22-25 dBm Alcance medio (puede haber interferencias)
7-10 metros 25-30 dBm Largo alcance (más lecturas duplicadas)

💡 Para uso electoral ONPE, se recomienda 1-2 metros para balance entre velocidad y precisión.

Configuración por TCP/IP (Red)

🔌 Requisitos de Red

  1. Conectar la antena RFID a la red local via cable Ethernet
  2. Configurar la PC en la misma red: 192.168.99.x
  3. IP de la antena (default): 192.168.99.202
  4. Puerto (default): 8889

✅ Verificación de Conexión

  1. Abrir terminal/CMD
  2. Ejecutar: ping 192.168.99.202
  3. Debe responder con tiempos < 10ms
  4. Si no responde, revisar:
    • Cable Ethernet conectado
    • Antena encendida
    • IP de la PC configurada correctamente

💡 Consejo: La conexión TCP/IP es más estable y rápida que USB. Se recomienda para uso en producción.

Configuración de la Aplicación

🔔 Sonidos de Notificación

Los sonidos se reproducen al escanear tags:

  • Sonido de éxito: Tags válidos y enviados correctamente
  • Sonido de error: Tags no identificados o errores de envío
  • Desactivar si hay muchas lecturas simultáneas

⏯️ Control Play/Pause

El botón en la esquina superior derecha controla el sistema:

  • ▶ Play: Sistema activo, acumula lecturas
  • ⏸ Pause: Ignora lecturas RFID
  • El sistema inicia en pausa por seguridad
  • Las lecturas pausadas se marcan en el historial

📊 Sistema de Acumulación

¿Cómo funciona?

  1. Al escanear el primer tag, inicia el contador
  2. Durante el intervalo configurado, acumula todos los tags detectados
  3. Al cumplirse el tiempo, envía el lote completo al servidor
  4. Se reinicia y espera la siguiente primera lectura

💡 Ventaja: Reduce tráfico de red y agrupa lecturas relacionadas.

🗑️ Envío Forzado

El sistema enviará inmediatamente si:

  • Se alcanza el máximo de tags configurado (default: 50)
  • Se presiona el botón "Datos de prueba"

Solución de Problemas

  • 🔴 La antena aparece desconectada
    • Verificar que la antena esté encendida
    • Comprobar cable Ethernet conectado
    • Verificar IP: ping 192.168.99.202
    • Revisar que la PC esté en red 192.168.99.x
    • Reiniciar la aplicación después de conectar
  • 📡 No detecta tags RFID
    • Verificar que el sistema esté en modo ▶ Play (no pausado)
    • Acercar el tag a la antena (distancia recomendada: 1-2 metros)
    • Reducir la distancia de lectura en configuración
    • Verificar que los tags sean UHF (no HF/NFC)
    • Revisar que el beeper esté activado para confirmar lecturas
  • ⚠️ Lecturas duplicadas excesivas
    • Reducir distancia de lectura (menor potencia)
    • Aumentar el umbral de duplicados (>200ms)
    • Configurar distancia a 1-2 metros
    • Alejar tags después de escanear
  • 🚫 Error al enviar al servidor
    • Verificar configuración de API en la pestaña correspondiente
    • Comprobar que la URL del servidor sea correcta
    • Verificar que el token de acceso sea válido
    • Revisar conexión a Internet
    • Consultar con el administrador del sistema

Información Técnica

🔧 Especificaciones del Lector

  • Modelo: UHF UR4 Fixed Reader
  • Frecuencia: UHF RFID (860-960 MHz)
  • Protocolo: EPC Class 1 Gen 2 / ISO 18000-6C
  • Potencia: 5-30 dBm (ajustable)
  • Rango: 0.1-15+ metros
  • Conexiones: TCP/IP (Ethernet), USB (Serial)

💻 Requisitos del Sistema

  • SO: Windows 10/11, Linux, macOS
  • Node.js: v14 o superior
  • RAM: 2GB mínimo
  • Red: Ethernet (para TCP/IP)
  • Puerto USB: Disponible (para modo USB)
